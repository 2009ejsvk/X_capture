# X / Threads Capture

X(트위터) 또는 Threads 게시물 링크를 붙여 넣으면 카드 형태로 미리보기하고 PNG로 저장하는 로컬 웹 앱입니다.

## 기능

### X 캡처
- `x.com` 또는 `twitter.com` 트윗 URL 불러오기 (`oEmbed`)
- `oEmbed` 실패 시 `vxtwitter` 보조 경로로 자동 재시도
- 작성자 프로필 이미지 자동 반영(실패 시 이니셜 폴백)
- 트윗 첨부 이미지 다중 표시(최대 4장)
- 다중 이미지 배치 옵션(나란히/세로)
- 작성자/핸들/날짜/본문 수동 편집
- 이미지 업로드(선택, 최대 4장)
- 모바일에서 고해상도 PNG 저장

### Threads 캡처
- `threads.com` 또는 `threads.net` 공개 게시물 링크 불러오기
- 로컬 Node 서버가 Threads 공개 HTML과 프로필 메타를 대신 읽어서 정규화
- 이미지 프록시(`/api/image`)를 통해 캡처 안정성 보강
- 작성자/핸들/날짜/본문/보조 메모 수동 편집
- 이미지 업로드(선택, 최대 4장)

## 실행

Threads 캡처까지 포함해 쓰려면 내장 Node 서버로 실행하세요.

```powershell
cd e:\X_capture
npm start
```

브라우저에서 `http://localhost:5173` 접속

X 캡처만 임시로 확인할 때는 정적 서버로 열어도 되지만, Threads 캡처는 동작하지 않습니다.

## 주의사항

- X/Twitter 정책 또는 API 응답 변경 시 URL 자동 파싱이 실패할 수 있습니다.
- 비공개 계정 트윗은 불러오지 못할 수 있습니다.
- 일부 트윗은 공식 `oEmbed`에서 404가 날 수 있어 보조 API를 사용합니다.
- 원격 이미지 CORS 정책에 따라 자동 썸네일 반영이 실패할 수 있으며, 이 경우 이미지 파일을 직접 업로드하면 됩니다.
- Threads는 공개 메타 기준으로 불러오므로 게시 시각이나 추가 이미지가 자동 추출되지 않을 수 있습니다.
- Threads 캡처는 로컬 API 서버가 필요합니다.
